gen5 khash [win & linux]
파일/폴더 통합 해싱과 전자 서명에 사용되는 단일 실행파일 툴입니다.
<경고 : 서명은 파일/폴더의 바이너리 내용은 보장하나, 인증서의 시각과 설명 등은 보장하지 않습니다.
인증서 생성일이나 서명을 같은 기능은 편의만을 위해 만든 기능이며 보안 목적으로는 권장하지 않습니다.>
<경고 : 인증서 내부에 공개키가 포함되어 있으며, 검증 시 이 값만 체크합니다.
공개키가 유효한지는 텍스트 파일을 열어 직접 확인하거나 다른 사용자 프로그램이 내부적으로 체크하게 하십시오.>

해싱과 전자 서명은 stdlib5.ksign을 사용하며,
CLI 환경에서의 파일 탐색기는 stdlib5.cliget을 사용합니다.
모드는 1~7이 존재하며, 이외의 숫자 입력 시 종료됩니다.

파일 해싱 (1)
cliget으로 해싱할 파일을 선택합니다.
해시값의 헥스(16진수 표시)와 Base64 인코딩 결과가 표시됩니다.

폴더 해싱 (2)
cliget으로 해싱할 폴더를 선택합니다.
해시값의 헥스(16진수 표시)와 Base64 인코딩 결과가 표시됩니다.
<경고 : 폴더 해시는 운영체제나 폴더의 복잡도에 따라 불안정할 수 있습니다.>

파일 서명 검증 (3)
파일의 내용이 바뀌었는지 인증서를 통해 검증할 수 있습니다.
먼저 cliget으로 인증서(sign.txt)를 선택합니다.
다음으로 검증 대상인 파일을 cliget으로 선택합니다.
먼저 인증서가 주장하는 이름과 서명과 같은 정보가 표시될 것입니다.
이 부분은 변조될 수 있으니 보안 목적으로는 다른 방법을 사용하세요.
다음으로 파일의 내용물이 올바른지 검증 결과가 표시됩니다.
이 부분은 변조될 수 없으니 보안 목적으로 사용하셔도 됩니다.

폴더 서명 검증 (4)
폴더의 내용이 바뀌었는지 인증서를 통해 검증할 수 있습니다.
먼저 cliget으로 인증서(sign.txt)를 선택합니다.
다음으로 검증 대상인 폴더를 cliget으로 선택합니다.
먼저 인증서가 주장하는 이름과 서명과 같은 정보가 표시될 것입니다.
이 부분은 변조될 수 있으니 보안 목적으로는 다른 방법을 사용하세요.
다음으로 폴더의 내용물이 올바른지 검증 결과가 표시됩니다.
이 부분은 변조될 수 없으니 보안 목적으로 사용하셔도 됩니다.

파일 서명 (5)
(7)에서 생성한 개인키로 파일에 전자서명을 할 수 있습니다.
먼저 cliget으로 개인키를 선택합니다.
개인키 비밀번호를 입력합니다. (생성 시 설정한 비밀번호입니다. 힌트가 표시될 겁니다.)
다음으로 서명할 파일을 cliget으로 선택합니다.
마지막으로 인증서 제작을 위한 추가 정보를 입력합니다. (보통 인증한 파일의 이름이나 간략한 용도 입력.)
프로그램과 같은 폴더에 인증서(sign.txt)가 생기면 파일과 인증서를 같이 배포하면 됩니다.

폴더 서명 (6)
(7)에서 생성한 개인키로 폴더에 전자서명을 할 수 있습니다.
먼저 cliget으로 개인키를 선택합니다.
개인키 비밀번호를 입력합니다. (생성 시 설정한 비밀번호입니다. 힌트가 표시될 겁니다.)
다음으로 서명할 폴더를 cliget으로 선택합니다.
마지막으로 인증서 제작을 위한 추가 정보를 입력합니다. (보통 인증한 폴더의 이름이나 간략한 용도 입력.)
프로그램과 같은 폴더에 인증서(sign.txt)가 생기면 폴더와 인증서를 같이 배포하면 됩니다.

RSA서명키 생성 (7)
사용자 전자서명을 위한 개인키/공개키를 생성합니다.
먼저 키 강도를 선택합니다. bit 단위이며 보통 2048을 사용합니다.
다음으로 서명의 이름을 입력합니다. 이름은 16바이트 이하여야 합니다.
영문은 글자당 1바이트이며 한글은 글자당 3바이트입니다.
RSA 키가 생성되면, 개인키를 암호화할 비밀번호와 비밀번호 힌트를 입력합니다.
개인키는 절대 남에게 공개되어선 안되기에 비밀번호로 잠가 놓아야 합니다.
모두 입력하면 프로그램과 같은 폴더에 공개키(public.txt)와 개인키(private.webp)가 있을 것입니다.
